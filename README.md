# concat-markdown
markdownファイル群を、50万文字のファイル25個以内になるように結合します。

# 実装完了報告：NotebookLM用Markdown結合スクリプト

指定されたディレクトリ内の `.md` ファイルを NotebookLM で扱いやすい形式に結合するスクリプト `concat_md.py` を作成しました。

## 実装内容

### [concat_md.py](file:///home/gaku/concat-markdown/concat_md.py)
- **再帰的検索**: サブディレクトリ配下の `.md` ファイルもすべて対象にします。
- **結合・分割ロジック**: 
  - 1ファイルあたりの最大文字数（デフォルト490,000文字）を管理。
  - 出力ファイル数を最大25個に制限。
  - 各ファイルの内容の冒頭に元のファイルパスをヘッダーとして追加。
- **柔軟な設定**: コマンドライン引数で入力/出力ディレクトリ、文字数制限、ファイル数制限をカスタマイズ可能です。

## 動作確認の結果

テスト環境を構築し、以下の条件で検証を行いました。
- **入力**: 10個のテスト用 `.md` ファイル。
- **条件**: 1ファイル最大200文字、最大5ファイルまで。
- **結果**: 
  - 期待通り、10個のファイルが5つの出力ファイルに分割されました。
  - 5ファイルの上限に達した際に適切に停止し、通知を表示しました。
  - 各出力ファイルに正しいソースヘッダーが含まれていることを確認しました。

### 実行例
```bash
python3 concat_md.py <入力ディレクトリ> <出力ディレクトリ>
```

## 使い方

1. スクリプトを実行します：
   ```bash
   python3 concat_md.py ./my_notes ./notebook_input
   ```
2. `./notebook_input` ディレクトリに `output_1.md`, `output_2.md` ... が生成されます。

> [!TIP]
> NotebookLMの50万文字制限に対し、デフォルトではヘッダー分を考慮して490,000文字に設定しています。必要に応じて `--max_chars` 引数で調整してください。
